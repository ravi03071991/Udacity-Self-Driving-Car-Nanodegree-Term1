As a first step, I decided to convert the images to brightness augmented images for Changing brightness to simulate day and night conditions. We will generate images with different brightness by first converting images to HSV, scaling up or down the V channel and converting back to the RGB channel.

Here is an example of a traffic sign image before and after brightness augmentation.

Brightness Augmentation

As a second step, I normalized the image data because …

The reason to do this in the process of training our network, we’re going to multiply these features (pixel values) with weights and add biases in order to cause activations that we then backpropogate with the gradients to train the model.

We’d like in this process for each feature to have a similar range so that our gradients don’t go out of control (and that we only need one global learning rate multiplier).

Another way you can think about it is deep learning networks traditionally share many parameters - if you didn’t scale your inputs in a way that resulted in similarly-ranged feature values sharing wouldn’t happen very easily because to one part of the image weight w is a lot and to another it’s too small.

Normalized Images

As a last step, I converted all the images to grayscale images. By doing this we will have only single channel in image which will be computationally efficient while classifying them.

Grayscale Images
